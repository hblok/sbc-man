## Development Skills & Standards

**Class Structure**
- Define all paths in a common class with @property methods returning pathlib.Path objects

**Python Style**
- Prefer custom classes and data structures over dictionaries
- Use pathlib instead of os.path
- Import modules, not classes (e.g., `import module` then `module.Class`, not `from module import Class`)
- Keep methods minimal; factor out logical sections into sub-methods
- Avoid importing inside methods when possible
- Keep classes small, preferably less than 400 lines

**Testing**
- Only one TestCase class per module
- Never document test methods
- Do not add main() or call unittest.main() in test modules
- Run tests after each significant change
- Maintain > 80% code coverage for math and core modules
- Use setUp() and tearDown() for test fixtures

**Documentation**
- Avoid verbose docstrings for simple methods
- Use single-line docstrings where appropriate
- Keep docstrings for complex logic only

**Delivery**
- Create a Pull Request against the main branch
